# 出力規則（最優先）

- 全ファイルをコードブロックで省略なく完全に出力すること。
- `// ...`、`// TODO`、`// 省略`、`// 以下略` は一切書かない。
- 全ファイルを出力し終えるまで要約・説明・感想を挟まない。

# 目的

React アプリ内で利用できる、軽量で拡張性の高い金融チャートコンポーネントを作成する。
ローソク足チャートとラインチャートを切り替え可能にし、テーマ変更、リサイズ、DPR対応、Tooltip、最新価格ライン、Y軸ラベル表示まで対応する。

SVG は使用せず、チャート本体は Canvas 2D のみで描画する。

# 技術前提

- React 19
- TypeScript strict (`noUnusedLocals` / `noUnusedParameters` / `erasableSyntaxOnly`)
- Vite
- Canvas 2D API
- d3-scale / d3-format / d3-time-format / d3-array は使用可（npm install 不要、インストール済み）
- d3 による DOM 描画は禁止
- CSS Modules は使わない
- 色はすべて theme から注入する
- renderer 内に固定色を書かない
- テストは `node:test` + `node:assert/strict` を使う（Vitest は未インストール）

# Canvas描画仕様

renderFinancialChart.ts に Canvas 描画全体を集約する。

描画順序は必ず以下。

1. clearRect
2. drawBackground
3. createScales
4. xTicks / yTicks 生成
5. drawGrid
6. chartType に応じて drawCandlesticks または drawLine
7. drawAxis
8. drawLatestPriceLine
9. drawLatestPriceLabel

# 背景

drawBackground:
- theme.background で全面塗りする。

# グリッド

drawGrid:
- plot領域内に縦横グリッドを描画する。
- xTicks / yTicks に基づいて線を引く。
- theme.grid を使う。
- 線は 0.5px 補正してぼやけを抑える。
- plot領域からはみ出さないようにする。

# 軸

drawAxis:
- X軸は下側。
- Y軸は右側。
- 軸線、目盛、ラベルを描画する。
- theme.axis / theme.text を使う。
- X軸ラベルは timeframeMs に応じて以下のフォーマットで表示を切り替える。
  - timeframeMs < 86_400_000 (1日未満, イントラデイ):
    - 通常のtick: `%H:%M:%S`
    - 日付が前のtickから変わった最初のtick: `%Y/%m/%d %H:%M:%S`
  - timeframeMs >= 86_400_000 (1日以上):
    - 日付が前のtickから変わった最初のtick: `%Y/%m/%d`
    - 同日内の重複tick: 空文字（ラベルを描画しない）
- Y軸ラベルは価格フォーマットする（`format(',.2f')` 相当）。

# ローソク足

drawCandlesticks:
- close >= open の場合は theme.candleUp
- close < open の場合は theme.candleDown
- ヒゲは line
- 実体は rect
- 実体の高さは最低 1px
- candleWidth は computeCandleWidth で算出する。
- plot領域外にはみ出さないよう clip を使う。

# ラインチャート

drawLine:
- close を結ぶ line chart を描画する。
- theme.line を使う。
- plot領域内に clip する。

# 最新価格ライン

drawLatestPriceLine:
- data の最後の close を最新価格とする。
- yScale(latest.close) に水平点線を描く。
- 点線は setLineDash([4, 4])。
- 描画後は setLineDash([]) に戻す。
- theme.latestPriceLine を使う。
- plot領域外なら描画しない。

# 最新価格ラベル

drawLatestPriceLabel:
- Y軸上、plot.right の右側に表示する。
- 背景は theme.latestPriceLabelBg。
- 文字は theme.latestPriceLabelText。
- 角丸矩形を描画する。
- ラベルは plot.top / plot.bottom 内に収める。
- 最新価格ラインより後、軸より後に描き、最前面に出す。

# スケール

createScales.ts:

- xScale は d3.scaleTime
- yScale は d3.scaleLinear
- x domain:
  - 空データ: [new Date(0), new Date(1)]
  - 1件: [time, time + 1ms]
  - 複数件: time の先頭〜末尾
- 未ソートデータの場合:
  - DEV環境では console.warn
  - domain は min/max で補正する
- y domain:
  - candlestick: low/high 全体
  - line: close 全体
  - 上下に 5% padding
  - 空データ: [0, 1]
  - 同値価格のみでも潰れないよう padding を入れる

computeCandleWidth:
- 隣接 time の xScale 差分の最小値を使う。
- 70% をローソク幅にする（`Math.floor(minStep * 0.7)`）。
- 最低 1px。
- 1件のみ、または有効な隣接ステップがない場合は 8px。

# HiDPI対応

setupHiDpiCanvas.ts を作る。

- `canvas.width = Math.round(width * devicePixelRatio)`
- `canvas.height = Math.round(height * devicePixelRatio)`
- `canvas.style.width = '${width}px'`
- `canvas.style.height = '${height}px'`
- サイズが変わらない場合は `canvas.width` / `canvas.height` を再代入しない（Canvas クリアを避けるため）。

FinancialChart 側では:

```
ctx.setTransform(devicePixelRatio, 0, 0, devicePixelRatio, 0, 0)
```

を使い、描画座標は CSS px 基準にする。

window が存在しない環境でも落ちないようにする。

# DPR変動追従

FinancialChart.tsx で devicePixelRatio を useState で管理し、変更時に再描画する。

実装パターン:
```typescript
const [devicePixelRatio, setDevicePixelRatio] = useState(() =>
  typeof window === 'undefined' ? 1 : window.devicePixelRatio || 1
);

useEffect(() => {
  if (typeof window === 'undefined') return;
  let mq = window.matchMedia(`(resolution: ${window.devicePixelRatio}dppx)`);
  const handler = () => {
    setDevicePixelRatio(window.devicePixelRatio || 1);
    mq.removeEventListener('change', handler);
    mq = window.matchMedia(`(resolution: ${window.devicePixelRatio}dppx)`);
    mq.addEventListener('change', handler);
  };
  mq.addEventListener('change', handler);
  return () => mq.removeEventListener('change', handler);
}, []);
```

devicePixelRatio は useEffect の依存配列に含める。

# リサイズ

useResizeObserver.ts:

- 親要素の `contentRect.width` / `contentRect.height` を ResizeObserver で監視する。
- 初回は `getBoundingClientRect()` でサイズを取得する（初回描画の1フレーム遅延を防ぐ）。
- 前回値と同じサイズなら setState を呼ばない（不要な再レンダーを防ぐ）。
- `ResizeObserver` がない環境でも落ちないようにする（try-catch は不要。`typeof ResizeObserver !== 'undefined'` で guard する）。

# Tooltip

Tooltip は Canvas に描かず、DOM overlay として canvas の上に重ねる。

useTooltipInteraction.ts:
- mousemove / mouseleave イベントを処理する。
- requestAnimationFrame で更新を間引く（rafIdRef + pendingPointerRef パターン）。
  - mousemove のたびに pendingPointerRef に最新座標を保存し、RAF が未予約なら予約する。
  - RAF コールバック内で pendingPointerRef を読んで setState する。
- plot外に出たら Tooltip を消す（setState(null)）。
- 既に null の場合は setState を呼ばない（`current ? null : current` パターン）。
- dataLength 変化時は index を `Math.min(index, dataLength - 1)` で clamp する（useMemo で処理）。
- getNearestDataIndexFromX を使う。
- 戻り値:
  - activeTooltipState: TooltipState | null
  - handleMouseMove: (event: MouseEvent<HTMLCanvasElement>) => void
  - handleMouseLeave: () => void

getNearestDataIndexFromX:
- pointerX を plot.left〜plot.right の比率に変換する。
- `Math.round(ratio * (dataLength - 1))` で index を求める。
- 範囲外は `Math.max(0, Math.min(index, dataLength - 1))` で clamp する。
- dataLength <= 1 は 0 を返す。

ChartTooltipOverlay.tsx:
- 縦点線ガイドを表示する（`borderLeft: 1px dashed theme.tooltipGuideLine`）。
- Tooltipパネルを表示する。
- theme.tooltipGuideLine / tooltipBackground / tooltipText / tooltipShadow を使う。
- 全要素に `pointerEvents: 'none'` を設定する。
- ガイド線の zIndex: 5、Tooltipパネルの zIndex: 10。
- Tooltipの横位置は以下の式で両端を制約する:
  ```
  const TOOLTIP_WIDTH = 180;
  const PADDING = 8;
  const OFFSET_X = 14;
  left = Math.max(PADDING, Math.min(width - TOOLTIP_WIDTH - PADDING, x + OFFSET_X))
  ```
- Tooltip の見た目（inline style で指定）:
  - background: theme.tooltipBackground
  - color: theme.tooltipText
  - boxShadow: theme.tooltipShadow
  - borderRadius: '6px'
  - padding: '8px 10px'
  - fontSize: '12px'
  - lineHeight: 1.5
  - minWidth: '156px'
  - fontFamily: 'ui-sans-serif, system-ui, sans-serif'
  - label と value を `justifyContent: 'space-between'` で左右配置
  - value は fontWeight 600
